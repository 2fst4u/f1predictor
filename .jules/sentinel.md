## 2025-05-03 - [CRITICAL] Prevent SSRF Bypass in URL Validation via String Matching
**Vulnerability:** A Server-Side Request Forgery (SSRF) bypass existed in `f1pred/web.py`'s `test_webhook` function. The validation check used `request.url.startswith("https://discord.com/api/webhooks/")`, which allowed an attacker to supply a malformed URL like `https://discord.com@127.0.0.1:80/api/webhooks/`. The `startswith` check would pass, but the underlying `httpx` request would interpret `discord.com` as the basic auth username and route the request to the internal loopback IP `127.0.0.1:80`, enabling internal network probing.
**Learning:** `startswith()` is inherently insecure for URL validation because URL semantics are complex (e.g., `<scheme>://<user>:<password>@<host>:<port>/<path>`). A string prefix match does not guarantee the host is actually what it appears to be.
**Prevention:** Always use proper URL parsing libraries (like `urllib.parse.urlsplit`) to explicitly extract and strictly validate individual URL components (`scheme`, `netloc`, `path`) against allowlists before performing server-side outbound requests based on user input.
